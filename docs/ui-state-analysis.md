### Страница «Анализ» — IncidentForm + результат

#### Текущее поведение полей формы

| Элемент | Идёт анализ | Не идёт анализ |
|---|---|---|
| Загрузка DOCX (drag&drop) | ✅ активно | ✅ активно |
| Все поля формы (title, description, date…) | ✅ активно | ✅ активно |
| Режим «Одна / Сравнить» | ✅ активно | ✅ активно |
| Выбор методики | ✅ активно | ✅ активно |
| Кнопка «Запустить анализ» | 🔄 крутится, disabled | ✅ активно |
| Кнопка «Найти похожие» | ✅ активно | ✅ активно |

#### Найденные проблемы

**🔴 P1: Форма не блокируется во время анализа**
- Пользователь может редактировать/стирать поля, пока идёт анализ
- Если случайно изменить поле — результат анализа не будет соответствовать тому, что на экране
- **Предложение:** пока `loading=true` — все поля формы заблокированы (disabled),
  загрузка DOCX тоже заблокирована. Только кнопка «Отустить анализ» видна как
  прогресс-индикатор.

**🔴 P2: Результат показывается вместе с формой**
- Форма и результат (ResultView / CompareView) показываются одновременно на одной странице
- Это приводит к длинной прокрутке и путанице: верхняя часть — поля, нижняя — результат
- После получения результата поля не обнуляются, но и не скрываются — непонятно,
  можно ли начать новый анализ
- **Предложение:** когда результат получен — форма скрывается, показывается только результат
  с кнопкой «Новый анализ» (которая обнуляет форму и возвращает к вводу).

**🟡 P3: Переход на Историю во время анализа**
- Пользователь нажимает «История» во время `loading=true`
- Анализ продолжается в фоне (App.jsx: handleSubmit не отменяется)
- Когда результат приходит — `page` уже 'history', но `result` сохраняется в стейт
- При возврате на «Анализ» — результат появляется, но **поля формы обнулены**
  (т.к. handleSubmit делал `setResult(null)` в начале, потом `setResult(data)`)
- **Фактический баг:** пользователь уходит на Историю → анализ завершается →
  возвращается на Анализ → видит результат, но поля пустые → непонятно, к чему результат
- **Предложение:** при уходе на другую вкладку во время анализа — показать предупреждение
  «Анализ ещё идёт, уверены что хотите уйти?» ЛИБО разрешить переход, но при возврате
  показать результат + контекст (что анализировалось).

**🟡 P4: Нет кнопки «Отменить анализ»**
- Пользователь запустил анализ, понял что ошибся — нельзя остановить
- **Предложение:** заменить кнопку «Анализирую…» на «Отменить» (AbortController для fetch)

**🟡 P5: Загрузка DOCX и анализ — две разные операции, но визуально слиты**
- DOCX-загрузка (заполнение полей) может занимать 1–2 минуты
- Анализ — ещё 1–3 минуты
- Во время загрузки DOCX поля тоже не блокируются — пользователь может
  менять то, что LLM только что заполнил
- **Предложение:** поля блокируются на время загрузки DOCX тоже

### Просмотр результата (из Истории)

Нажатие на карточку в Истории → переключает `page` на 'view', показывает ResultView
или CompareView с кнопкой «← Назад в историю».

**🟡 P6: Нет способа из просмотра результата вернуться к Анализу**
- Только «Назад в историю» — нет кнопки «Новый анализ»
- **Предложение:** добавить кнопку «Новый анализ» в режим просмотра из Истории

## 2. Сводка предложений

| # | Проблема | Решение | Приоритет |
|---|---|---|---|
| P1 | Поля формы активны во время анализа | `disabled={loading \|\| uploading}` на все поля | 🔴 высокое |

| P5 | Поля активны при загрузке DOCX | `disabled={loading \|\| uploading}` | 🔴 высокое (вместе с P1) |
| P6 | Нет кнопки «Новый анализ» из просмотра | Добавить кнопку | 🟡 среднее |
| P7 | Поиск похожих во время анализа | `disabled={loading}` | 🟢 низкое |

## 3. Предлагаемая модель состояний

```
┌─────────────┐     «Запустить анализ»     ┌──────────────┐
│   ВВОД       │ ─────────────────────────→ │   АНАЛИЗ     │
│   (форма)    │                             │  (спиннер)   │
│   поля: ✏️   │                             │  поля: 🔒    │
│   кнопка: ▶  │                             │  кнопка: ⏹   │
└─────────────┘                             └──────┬───────┘
       ↑                                           │
       │ «Новый анализ»                            │ результат получен
       │                                           ↓
       │                                    ┌──────────────┐
       └─────────────────────────────────── │  РЕЗУЛЬТАТ   │
                                            │  (ResultView │
                                            │   Compare)   │
                                            │  кнопка: ➕   │
                                            └──────────────┘
```

**Три чётких состояния:**
1. **ВВОД** — форма активна, результата нет
2. **АНАЛИЗ** — форма заблокирована, кнопка «Отменить», спиннер
3. **РЕЗУЛЬТАТ** — форма скрыта, показан результат, кнопка «Новый анализ»
