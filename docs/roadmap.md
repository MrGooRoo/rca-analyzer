## 🔧 К реализации

### 1. Управление провайдерами (админ)
- [ ] Админ выбирает провайдера (OpenRouter, OpenModel и т.д.)
- [ ] Настройка API-ключа и базовых параметров провайдера
- [ ] Система автоматически считывает каталог моделей провайдера

### 2. Категоризация моделей
- [ ] Авто-группировка моделей по размеру контекста:
  - **Полное** — контекст ≥64K
  - **Сбалансированное** — контекст 16K–63K
  - **Экспресс** — контекст <16K
- [ ] Модели отображаются с полосой контекста (визуализация)

### 3. Выбор моделей (пользователь)
- [ ] Экран выбора модели перед загрузкой DOCX
- [ ] В каждой группе (Полное / Сбалансированное / Экспресс) — radio button
- [ ] Индикация: бесплатно (зелёный) / платно (оранжевый)
- [ ] Предупреждение о списании при выборе платной модели

### 4. Личный кошелёк (заглушка)
- [ ] Отображение баланса в UI пользователя
- [ ] Заглушка: баланс $0.00, списания не производятся
- [ ] В будущем: зачисление средств, списание за платные модели

### 5. Верификация DOCX (админ-опция)
- [ ] В админ-панели переключатель «Верификация DOCX» (on/off)
- [ ] При включении — после извлечения DOCX вызывается verifier_model
- [ ] Проверка: не пропущены ли факты, корректен ли тип/тяжесть
- [ ] Пользователь не видит этот процесс — это внутренняя проверка
- [ ] Влияет на время и стоимость (2× токены)
